üî¨ Latent Space Visualization

This 3D plot shows how the autoencoder compresses transaction data into a lower-dimensional latent space. Normal transactions cluster together while anomalous (potentially fraudulent) transactions appear as outliers. The visualization helps understand how the model learns to distinguish between normal and suspicious patterns.

üß† How This Works

Step 1 - Data Compression: The autoencoder takes all the transaction features (like amount, time, customer age, etc.) and squeezes them down into just 16 numbers called "latent dimensions" using its encoder part. Think of it like taking a big, detailed picture and making a smaller, simpler version that still captures the important parts.

Step 2 - Smart Dimension Picking: Instead of just using the first 3 dimensions (which might be boring and all the same), we look at all 16 dimensions and pick the 3 that have the most interesting differences between transactions. It's like choosing the 3 most colorful crayons from a box of 16 to draw the most interesting picture.

Step 3 - Making It Pretty: We then adjust the numbers so they're all on the same scale (like making sure all the colors are equally bright), which makes the 3D picture much easier to understand and more balanced.

Step 4 - Finding Patterns: Normal transactions (shown in green) tend to group together in clusters, while suspicious transactions (shown in red) stand out as outliers. This happens because the autoencoder learned during training that normal transactions should be easy to reconstruct, while unusual ones are harder to predict.

When This Happens: This visualization is created right after the autoencoder processes the transaction data through its encoder layer, but before it tries to reconstruct the data. It's like taking a snapshot of how the model "sees" each transaction in its simplified, compressed form.
